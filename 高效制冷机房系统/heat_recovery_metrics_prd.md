# 回收热量与热回收价值指标 PRD

## 目标
说明“回收热量”和“热回收价值”指标的计算方式、计算频率与汇聚维度，作为前后端口径一致的依据。

## 指标定义
### 1) 回收热量
- 指标来源：大数据平台后台计算
- 计算频率：每小时计算一次
- 汇聚维度：自动汇聚至 **天 / 月 / 年**
- 说明：前端仅展示后台计算结果，不在前端重复计算

### 2) 热回收价值
- 指标来源：大数据平台后台计算
- 计算公式：
  - `热回收价值 = 回收热量 × 热单价`
- 计算频率：每小时计算一次
- 汇聚维度：自动汇聚至 **天 / 月 / 年**（热回收价值维度）
- 说明：前端仅展示后台计算结果，不在前端重复计算

## 数据口径
- **回收热量** 与 **热回收价值** 均以后台计算为准
- 若后台未返回数据，前端展示 `--`

## 展示规则
- 时间粒度与页面筛选粒度保持一致（天 / 月 / 年）
- 单位与后台返回一致（如：GJ、万元/元）

## 异常与边界
- 当任一指标缺失或为 0：
  - 仍展示该指标，但值为 `0` 或 `--`（按后台返回规则）
- 当热单价为阶梯计费：
  - 前端不展示热单价，仅展示计算后的热回收价值

## 计入节费点击后的级联计算逻辑
### 触发条件
- 用户点击“计入节费”开关/按钮

### 级联计算口径
- **不计算水费**
- **默认按高效制冷机房成本逻辑**（EER 维度）
  - `baseline_cost = cost(5.0)`（基线 EER=5.0 的电费成本）
  - `actual_cost = cost(eer_actual)`（实际 EER 的电费成本）
  - `electric_saving = baseline_cost - actual_cost`
  - 若 `electric_saving < 0`，按 0 计

### 与“回收热量 / 热回收价值”的级联关系
- **回收热量**：始终来自后台小时计算并自动汇聚结果，前端不参与计算
- **热回收价值**：后台按 `回收热量 × 热单价` 计算并汇聚，前端不参与计算
- **总节省费用**（计入节费开启后）：
  - `total_saving = electric_saving + heat_recovery_value`
  - 若 `heat_recovery_value` 缺失，则 `total_saving = electric_saving`

- 切换后刷新相关指标卡、图表与明细列表
- 统计粒度与页面筛选粒度保持一致（天 / 月 / 年）
